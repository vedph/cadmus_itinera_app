Sub-Models

In our data design, several models share some sub-models. For instance, whenever a structured person name occurs (as a correspondent in a letter, as an actor in the history of a manuscript, etc.), this can be of type PersonName.

As data is modular, so is the editing system. In this case, we provide an editor for each sub-model, which can be included into a higher-order editor, thus composing its UI.

You can see the sub-models editors page for a live demonstration.
